# Contributing to Frog Hill Labs

Thanks for taking an interest in a Frog Hill Labs project.

Most repositories here document practical software, electronics, radio, automation, infrastructure, and reverse-engineering work. Contributions that make a project clearer, sturdier, safer, or more useful are welcome.

## Before making a change

- Read the repository README and any project-specific documentation.
- Check existing issues and pull requests to avoid duplicating work.
- For substantial changes, open an issue first so the approach can be discussed.
- Never include credentials, private keys, tokens, personal data, or private infrastructure details.

## Pull requests

Keep pull requests focused and explain:

- What changed
- Why the change is useful
- How it was tested
- Any limitations, risks, or follow-up work

Update documentation and tests when the change affects behavior. Small, understandable changes are preferred over large unrelated batches.

## Project-specific guidance

A repository may define additional contribution requirements. When it does, those instructions take precedence over this organization-wide default.

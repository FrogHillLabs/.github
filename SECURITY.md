# Security Policy

Frog Hill Labs projects may include experimental software, self-hosted integrations, device tooling, and reverse-engineering work. Unless a repository states otherwise, projects should not be assumed to have undergone a formal security audit.

## Reporting a vulnerability

Please do not disclose suspected vulnerabilities, credentials, private infrastructure details, or exploit instructions in a public issue.

Use the repository's **Security** tab and **Report a vulnerability** option when private vulnerability reporting is available. Otherwise, use the contact information published by the repository or at [froghilllabs.com](https://froghilllabs.com).

If no private contact method is available, open a public issue requesting a private security contact, but do not include sensitive technical details.

When reporting, include enough information to reproduce and assess the problem, along with any known impact or suggested mitigation.

## Supported versions

Support status is documented by each repository. Experimental or archived projects may not receive security fixes.
